# Email access is product-scoped delegation, not identity linking

Status: accepted 2026-07-09. A verified email Principal may explicitly authorize a distinct Agent Principal to exercise that email's grants inside one Finite product, with product-owned revocation and audit; the agent continues signing as itself, Finite Identity never turns the authorization into a Principal Link or NIP-05 binding, and Finite Brain must separately issue the agent the Folder Key Grants required to decrypt content.
