## Cutover checklist â€” re-register against finitecomputer/finite-mono

Runners are repo-scoped. For mono CI to build images on this box, each
runner (or a new runner) must be registered against
`finitecomputer/finite-mono`. Per runner:

1. Get a registration token: repo Settings â†’ Actions â†’ Runners â†’ New
   self-hosted runner (or `gh api -X POST
   repos/finitecomputer/finite-mono/actions/runners/registration-token`).
   Tokens are short-lived; never write one to disk or this repo.
2. Stop and unregister the old registration (as `ubuntu`, in the install dir):

   ```sh
   sudo ./svc.sh stop && sudo ./svc.sh uninstall
   ./config.sh remove --token <REMOVAL_TOKEN_FROM_OLD_REPO>
   ```

3. Re-register. **Preserve the exact custom labels** â€” finitechat's existing
   workflows select runners by `hermes-runtime` (and `docker`,
   `finite-lat-2`); migrated workflows in finite-mono must keep matching.
   Exact shape (placeholders, never real tokens):

   ```sh
   ./config.sh \
     --url https://github.com/finitecomputer/finite-mono \
     --token <REGISTRATION_TOKEN> \
     --name <RUNNER_NAME> \                # e.g. finite-lat-2-finitechat-hermes-runtime
     --labels <CUSTOM_LABELS> \            # e.g. finite-lat-2,docker,hermes-runtime
     --work _work \
     --unattended
   sudo ./svc.sh install ubuntu && sudo ./svc.sh start
   ```

   Per-runner `--labels` values:
   - finitechat-hermes-runtime: `finite-lat-2,docker,hermes-runtime`
   - finitecomputer-tinfoil-runtime: `finite-lat-2,nix,docker,tinfoil-runtime`
