## What to build

Carry the Dashboard-Aligned Product Theme through the complete knowledge
workspace so Files, Search, Page reading/editing, and Graph View feel like one
native Finite surface while retaining every current interaction and information
structure.

## Acceptance criteria

- [ ] Files and Search trees, hierarchy, selection, expansion, empty states, counts, and toolbars use the dashboard-aligned visual language in both themes.
- [ ] Page reading, visual/source editing, save state, code content, links, metadata, and editor chrome are fully rethemed without behavioral changes.
- [ ] Context menus, command palette, slash menu, disclosures, popovers, and related transient states are cohesive and legible in both themes.
- [ ] Graph View nodes, links, labels, filters, controls, replay, overlays, statistics, and empty states are cohesive and legible in both themes.
- [ ] Existing Files, Search, Page, editor, command, context-menu, Graph View, and replay behavior remains intact through the same controls and DOM hooks.
- [ ] Representative resumed-session knowledge workspace states are visually verified at desktop width in light and dark modes.
- [ ] Targeted Product Client tests and the seeded verifier pass.
